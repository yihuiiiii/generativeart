# 台南市東區 防災路徑分析系統 - 使用說明

## 新增功能

### 1. GPS 定位功能
- **位置**: 側邊欄「功能選項」區域的「🔴 啟動 GPS 定位」按鈕
- **功能**: 
  - 點擊按鈕啟動GPS定位，您的實時位置將顯示在地圖上（藍色圓點）
  - 圓點旁邊的虛線圓圈代表GPS精度範圍
  - 按鈕會改變顏色並顯示「關閉 GPS 定位」以表示GPS已啟動
  - 只需打開此HTML檔案，瀏覽器會自動請求位置權限
- **注意**: 瀏覽器會請求位置權限，請點擊「允許」

### 2. 匯入收容所資訊功能
- **位置**: 側邊欄「功能選項」區域的「📍 匯入收容所資訊」按鈕
- **功能**: 
  - 點擊按鈕打開輸入框
  - 輸入JSON格式的收容所資訊（見下方範例）
  - 系統會自動將新收容所添加到地圖和下拉選單

### 3. 圖例更新
- **避難收容所**: 房子icon（房子內有跑步的人）
- **救濟站**: 紅色十字icon
- **防空疏散避難設施**: 防空洞icon

### 4. 路線樣式更新
- **一般最短路線**: 橘色長虛線（距離優先）
- **災難安全路線**: 紅色實線（推薦，風險最低）

### 5. 收容所詳細資訊
- 點擊地圖上的任何icon即可查看該收容所的完整資訊，包括：
  - 設施名稱
  - 地址
  - 設施類型
  - 適合的災難類型
  - 收容人數（室內/室外分開顯示）
  - 建築物資訊
  - 服務里別

## 匯入收容所資訊 - JSON 格式範例

```json
{
  "name": "示例避難收容所",
  "addr": "台南市東區崇德路100號",
  "lat": 23.0050,
  "lng": 120.2098,
  "indoor": 500,
  "outdoor": 200,
  "type": "shelter",
  "disasters": ["earthquake", "flood"],
  "note": "RC結構建築",
  "serves": "示例里別"
}
```

### JSON 字段說明：
- **name** (必填): 收容所名稱
- **addr**: 地址
- **lat** (必填): 緯度（例如：23.0050）
- **lng** (必填): 經度（例如：120.2098）
- **indoor**: 室內收容人數
- **outdoor**: 室外容量人數
- **type**: 設施類型，可選值：
  - `shelter` - 官方避難收容所（紅色房子icon）
  - `relief` - 救濟站學校（藍色十字icon）
  - `civil` - 防空疏散避難設施（灰色防空洞icon）
- **disasters**: 適合災難類型陣列，可選值：
  - `earthquake` - 地震
  - `flood` - 水災
  - `war` - 戰爭
- **note**: 建築物資訊備註
- **serves**: 服務里別

## 如何從 Google Maps 獲取收容所座標

1. 打開 Google Maps 我的地圖：https://www.google.com/maps/d/u/0/viewer?mid=1tGAKrgll78TtWGYrEE450OKECwF-Cks
2. 點擊地圖上的任何標記，記下其名稱和地址
3. 右鍵點擊該位置→「這是哪裡？」，獲取座標
4. 座標格式為 `緯度,經度`
5. 將該座標拆分為 lat 和 lng
6. 按照上方JSON格式組合，粘貼到匯入框中

## 使用流程

1. **打開HTML文件**：所有避難設施已預先標記在地圖上
2. **啟動GPS定位**（可選）：點擊「🔴 啟動 GPS 定位」按鈕
3. **選擇災難類型**：在「災難類別」中選擇「地震」、「水災」或「戰爭」
4. **設定路徑**：
   - 點擊「點擊地圖設定出發位置」，然後在地圖上點擊出發地點
   - 或從「選擇目標避難設施」下拉選單中選擇目的地
5. **執行路徑分析**：點擊「執行路徑分析」按鈕
6. **查看結果**：系統會顯示兩條路線：
   - 一般最短路線（距離優先）
   - 災難安全路線（風險最低，推薦）

## 匯入新收容所

1. 點擊「📍 匯入收容所資訊」按鈕
2. 在彈出框中輸入JSON格式的收容所資訊
3. 點擊確定
4. 新收容所將自動添加到地圖和選單中
5. 現在可以使用新收容所作為路徑分析的目標

## 說明

- 所有功能都在客戶端本地運行，無需伺服器
- GPS定位需要瀏覽器權限和HTTPS連線（本地文件可能受限）
- 新增的收容所存儲在記憶體中，刷新頁面後會重置
